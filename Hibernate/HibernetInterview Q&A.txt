-Advantages of Hibernate over JDBC
	Eliminates a lot of boiler plate code
	Supports inheritance,associations and collections
	HQL is more object-oriented and closer to java
	Implicitly provides transaction management
	It wraps exceptions and throw JDBC exceptions which are the unchecked exceptions.
-Different functionalities supported by Hibernate
	It is an ORM tool
	It uses HQL
	It supports auto DDL operations
	It also has an auto primary key generation support
	Supports cache memory
	Exception handling is not mandatory
-Hibernate Configuration file are Dialect information so that hibernate knows the database type and mapping file or class details.
-Hibernate Annotations
-Data validation
	Data validation is used in the presentation layer with the use of java script and the server-side code before processing.
-Different states of a persistent entity?
	Transient	: is not associated with the Session and has no representation in the database
	Persistent 	: you can make a transient instance persistent by associating it with a Session
	Detached 	: If you close the Hibernate Session the persistent instance will become a detached instance.
