-Spring the Big Picture
	Spring could mean Spring boot
	Spring data
	Spring cloud
	Spring batch
-Spring cloud was built on top of Spring boot and simplified the development of applications that make use of distributed architectures such as micro services.
(Service discovery and configuration).
-Spring started as a response to the complexity of J2EE and today Spring is so much more than an alternative to Java EE.
-Spring makes it easy to create Java enterprise applications.
-Spring is flexible,modular and backwards compatible.
-Spring Boot Makes Getting Started with Spring Quick and Easy.
-Spring boot auto-configuration
	attempts to automatically configure your Spring application based on the dependencies that you have added.
	Auto-configurations are Contextually Aware and Smart
	Setting up Auto-configurations is Effortless
	@EnableAutoConfiguration
	public class DemoApplication {
		...
	}
-Spring boot makes it easy to create stand-alone, production-grade, Spring-based applications that you can 'just run'.
-Takes an opinionated view of building production-ready Spring applications.

-Understanding Spring's Foundations The Spring Framework
-Software Framework
	is a universal,reusable software environment that provides particular functionality as part of a larger software platform to facilitate development of software applications, products and solutions.
-Application -> (Spring Projects + Spring Framework)
-Spring framework six key areas:
	Core,Web,APO,Data Access,Integration,Testing
-Dependency Injection
	The object fulfills its own dependencies
	The object declares what it depends on and something else fulfils the dependency.
-Spring core is a dependency injections container.
	Creates and maintains objects and their dependencies
	Less for developer to manage
-Spring Web
	Framework for handling web requests
	Spring Web MVC
	Spring Web Webflux
-Java servlet
	A servlet is an object that receives a request and generates a response based on that request.
-A Standard Request Using Spring Web MVC
	Request -> Web Server -> Servlet API -> Spring Web MVC -> Business Logic 
-Spring Webflux
	Reactive Programming
	A reactive programming paradigm concerned with data streams and the propagation of change.
	React rather than wait.
	A different way of handling web requests.
	Asynchronous execution and don't block (wait) 
	Better resource utilisation
-Spring AOP
	A programming paradigm that aims to increase modularity by allowing the separation of cross-cutting concerns.
	A way of programming that increases organization of code for concerns that span multiple tiers of an application.
	Used to implement features in Spring
	A valuable tool for developers to handle cross-cutting concerns
-Spring Data Access
	A transaction is a general unit of work that must happen together or not all all
-Spring Integration
	Integration is all about making different systems and applications work together.
	How do you expose operations to other systems.
	How do you invoke or run operations on other systems.
	RMI (Remote Method Invocation),Messing System,Web Services
-Spring Testing
	Dependency Injection is about dealing with the way objects fulfil their dependent objects
	Spring Framework comes with several built-in mocks to make testing easier and faster.
	Integration testing in software testing in which individual software modules are combined and tested as a group.It occurs after unit testing.
-A distributed system is just a number of different applications all linked together by a network (micro services)
-Spring Cloud
	Spring cloud provides tools for developers to quickly build some of common patterns in distributed systems.
	Distributed systems is just a number of different applications all linked together by a network.
-Spring Security
	@EnableWebSecurity
	public class SecurityConfig extends WebSecurityConfiguerAdapter {
		
		@Override
		protected void configure(HttpSecurity http) throws Exception {
			http.
				authoizeRequests()
				.antMatchers("/css/**","/index").permitAll()
				.antMatchers("/user/**").hasRole("USER")
				.and()
				.formLogin()
				.loginPage("/login")
				.failuerUrl("/login-error");
		}
		
		@Autowired
		public void configureGlobal(AuthenticationManagerBuilder auth) throws Exception {
			auth.
				inMemoryAuthentication()
				.withUser("user")
				.password("password")
				.roles("USER");
		}
	}
