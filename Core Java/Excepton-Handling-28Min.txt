-What design patterns is used to implement Exception handling features in most languages?
	Chain of responsibility pattern defined as a way of passing a request between a chain of objects. 
-Code in finally is NOT executed  only in two situations:
	If exception is thrown in finally.
	If JVM crashes in between
-Try without catch is allowed.
-Below class definitions show the pre-defined exception hierarchy in java
	class Error extends Throwable{}
	class Exception extends Throwable{}
	class RuntimeException extends Exception {}
-Below class definitions show creation of a programmer defined exception in java.
	class CheckedExceptoin1 extends Exception {}
	class CheckedException2 extends CheckedException1{}
	
	class UnCheckedException extends RuntimeException {}
	class UnCheckedException2 extends UnCheckedException {}
-Checked exceptions have to be handled or thrown
-Error:	Error is used in situations when there is nothing a programmer can do about an error.
	Ex. StackOberflowError, OutOfMemoryError.
-Exception: Exception is used when a programmer can handle the exception.
-Unchecked Exception: RuntimeException and classes that extends RuntimeException are called unchecked exception.
-Checked Exception: Other exception classes
-try with resources
	When the try block ends the resources are automatically released.
	try (BufferReader br = new BufferReader(new FileReader("FILE_PATH"))) {
		String line;
		while((line = br.readLine() != null)){
			System.out.println(line);
		}
	}catch(IOException e){
		e.printStackTrace();
	}
-try-with resources is available to any class that implements the AutoCloseable interface
	public interface AustoClosable {
		void close() throws Exception;
	}
